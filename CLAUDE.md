# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Korean-language tool that reads customer interviews from Notion, extracts marketing insights via Claude API, and generates an interactive HTML dashboard. Designed for non-technical users running customer discovery interviews.

## Commands

```bash
python3 setup.py              # Interactive first-time setup (generates config.py)
python3 main.py --force       # Full analysis of all interviews (3-5 min, ~19 API calls)
python3 main.py               # Incremental — only re-analyzes changed interviews
python3 build_dashboard.py    # Rebuild dashboard HTML only (no API calls)
python3 watch.py              # One-shot: check Notion for changes, analyze if needed, deploy to Firebase
firebase deploy --only hosting # Manual Firebase deploy
```

## Architecture

```
Notion DBs → notion_reader.py → main.py → insight_extractor.py (Claude API)
                                    ├──→ notion_writer.py (write back to Notion)
                                    └──→ build_dashboard.py → index.html
```

**main.py** orchestrates: read interviews → extract individual insights (1 API call per interview) → extract common insights (1 synthesis call) → write to Notion → build dashboard.

**Caching**: `.insight_cache.json` stores insights keyed by page ID + `last_edited` timestamp. Incremental runs skip unchanged interviews.

## Key Module Responsibilities

- **notion_reader.py**: Recursively reads all Notion block types (including code blocks marked as `[답변]` for interview answers). Handles pagination (100+ blocks), rate limiting (0.35s/request), retry with backoff, and title extraction from tables when page titles are just emojis.
- **insight_extractor.py**: Two prompt levels — individual (JSON output with pain/value/voice/price/trigger/objection) and common (markdown with ICP, USP flows, messaging matrix, copy kit, objections, before/after, pricing). Uses `PRODUCT_NAME`, `PRODUCT_DESC`, `TARGET_CUSTOMER` from config.
- **build_dashboard.py**: Parses Claude's markdown output (tables via `_tbl()`, sections via `_sec()`). Injects Notion page URLs into every quote for source attribution. Replaces `__DASHBOARD_DATA_PLACEHOLDER__` in `dashboard.html` template.
- **notion_writer.py**: Deletes old insight blocks (preserves databases/child pages), appends new blocks in batches of 50.
- **watch.py**: Compares `last_edited` timestamps against `.last_state.json`, runs `main.py` + `firebase deploy` only on changes.

## Config (config.py, gitignored)

Generated by `setup.py`. Contains: `NOTION_TOKEN`, `CLAUDE_API_KEY`, `PAGE_ID`, `DATABASES` dict, `EXCLUDE_PAGE_IDS` list, `PRODUCT_NAME`, `PRODUCT_DESC`, `TARGET_CUSTOMER`. All databases must be `child_database` blocks on the single `PAGE_ID` page.

## No External Dependencies

Everything uses Python stdlib (`urllib.request`, `json`, `re`, `time`, `subprocess`). No pip install needed.

## Notion Block Reading Gotchas

- Interview answers are often in **code blocks** (not paragraphs) — these get `[답변]` prefix
- Pages with emoji-only titles need fallback title extraction from table rows (participant name)
- Synced blocks, column lists, toggles, and child pages are read recursively up to depth 5
- Rate limit: Notion API enforces 3 req/sec; code enforces 0.35s minimum between calls

## Dashboard Data Flow

Individual insights (JSON per interview) → aggregated into `raw_pains` array with source/segment/notion_url → common insights parsed into structured objects (ICP, USPs, matrix, copy kits) → all injected as single JSON blob into `dashboard.html` template → saved as `index.html`.
